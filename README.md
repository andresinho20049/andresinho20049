## [Rest API with Oauth2 authentication and documented using Swagger](https://github.com/andresinho20049/spring-oauth-boilerplate)
This Spring boot project was developed with the default authentication settings in mind and documentation to serve as a basis for developing other projects.

About:
 - Role-based access control with JWT
 - technology used
    - Java 8
    - Spring Boot 2.7.0
    - Spring JPA latest
    - Spring Security 5.6.4     
    - Spring Secutiry Oauth2 Autoconfigure 2.1.5
    - Springfox (Swagger) 3.0.0
    - H2 Database - test basis
    - Lombok - <a href="https://projectlombok.org/setup/eclipse" target="_blank">Help setting up lombok</a>
    - Log4J
    - Project Maven
- Application.properties - Default
    - Port: 5000
    - Profile: dev
    - Base path: /api
    - Encrypt: bcrypt
    - Hibernate DDL: update
- ExceptionHandler
    - ProjectException:
        - Status: 400
        - Description: Exception provoked, n reasons, but mainly business rule
    - AuthorizationException:
        - Status: 403
        - Description: Access Denied with RuntimeException

### Getting Started
1. Git clone project
 ```git
    git clone https://github.com/andresinho20049/spring-oauth-boilerplate
 ```

2. Inside the project folder, run
```mvn
    mvn clean && mvn install package
```

> In this example I will consider using Eclipse, but just adapt to your use, ok.

3. This project uses lombok, if you haven't already installed this plugin in your eclipse, click:
Help > Install new software...
In the work with field, paste: https://projectlombok.org/p2
Check lombok and then finish

4. Optional step: Click Project > Update maven project > Ok

5. Press start, if you prefer, paste this command in your terminal
```mvn
mvn package spring-boot:run 
```

  ## Libraries used
  I'll talk a little about the tools/libraries used, and explain why they were developed that way.

  ### Material UI
  Material UI is a library of React UI components that implement Google's Material Design.

  It includes a comprehensive collection of ready-to-use pre-built components.

  Material's interface is beautiful and features a set of customization options that make it easy to implement a custom interface system.

  ### Axios
  Axios is a Promises-based HTTP client for making requests. Axios brings some advantages and simplifications, being better to use it than just Fetch, such as simple configuration, automatic JSON conversion, Interceptors, etc...

  ### Unform
  Unform, forms API, advantage of using it because it does not use state, so when it is typed in the input, the other inputs do not suffer rendering.

  ### Yup
  Yup is a JavaScript schema builder for schema parsing and validation.

  ### CryptoJs
  CryptoJs, as the name implies, is a cryptography library, allowing the use of ALGORITHM and Secret Key, used in the end-to-end Encrypt application.

  ### React Router Dom
  React Router allows for "client-side routing".

  React is a popular library for building single-page applications (SPAs) that render client-side.

  and unlike conventional multi-page applications, browsing these views should not result in the entire page reloading.

  ## Security
  This React app consumes service from a Java Spring app with Oauth2 Authentication, and the login scheme must follow the validation and authentication requirements of the API.

  ### Front security layer
  As mentioned before, the back-end works with Authentication Server Oauth2, below I detail how the login works, in short, the session will be validated through the jwt token, if it is expired or non-existent, the user will be logged out.

  > Request: `oauth/token`
  ```ts
      const encryptedPassword = encryptedAES(login.password);
      const encryptedKeyVerify = encryptedAES(process.env.REACT_APP_KEY_VERIFY || '');
      const passwordEncoded = Buffer.from(`${encryptedPassword}|${encryptedKeyVerify}`).toString('base64');

      const data = new URLSearchParams({ 'username': login.username });
      data.append('password', passwordEncoded);
      data.append('grant_type', 'password');

      const config = {
          method: 'post',
          url: `${process.env.REACT_APP_BASE_URL}/oauth/token`,
          withCredentials: true,
          auth: {
              username: process.env.REACT_APP_AUTH_USERNAME || '',
              password: process.env.REACT_APP_AUTH_PASSWORD || ''
          },
          headers: {
              'Content-Type': 'application/x-www-form-urlencoded',
          },
          data: data
      };

      const res = await axios(config);
  ```

  > The username and password informed in the body, refers to the client (user) that is logging in, the clientId and clientSecret informed in the auth block serves for the application_client to authenticate itself with the Authentication server.
